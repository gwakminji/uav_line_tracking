# UAV Line 프로젝트 실험 요약

## 1. 프로젝트 목표
이 프로젝트는 GPS가 없는 환경에서 카메라만으로 UAV의 위치를 추정하고, 선형 구조(라인/격자/교차점)를 이용해 이동 경로를 추적하는 것을 목표로 진행되었다.

핵심 아이디어는 다음과 같다.
- 카메라 영상에서 라인을 검출한다.
- 검출된 라인의 기하학적 특징(기울기, 길이, 교차 여부, 중심 위치)을 이용해 위치를 추정한다.
- 필요할 경우 ArUco 마커를 보조 지표로 사용해 시각적 기준점을 확보한다.

---

## 2. 초기 실험: 흑백 기반 라인 검출
처음에는 흑백 이미지 기반의 단순한 라인 추적 실험부터 시작했다.

### 진행 내용
- grayscale 기반 이진화 실험
- Global / Adaptive / Otsu 방식 비교
- Canny edge + Hough 변환으로 직선 검출
- HoughLinesP를 이용한 선분 검출 실험

### 주요 포인트
- 흑백 기반 방식은 구현이 단순하고 빠르다는 장점이 있었지만,
- 라인과 배경의 밝기 차이가 작은 상황에서는 안정성이 떨어졌다.
- 위 원인으로는 라인이 단순히 "밝기 차이"로 구분되는 환경에서는 한계가 있었다.

이 단계는 이후 컬러 기반 접근으로 이어지는 기초 실험이었다.

---

## 3. 컬러 기반 실험으로 전환
흑백만으로는 한계가 있어서, 라인의 대비가 더 잘 보이는 컬러 기반 접근으로 바꾸었다.

### 전환 이유
- 라인이 색상 차이에 의해 더 명확하게 보이는 경우가 있었다.
- 배경과의 대비가 색상 축에서 더 분명하게 나타나는 상황이 있었다.
- 밝기 기반 방식보다 색상 기반 방식이 더 강건하게 동작할 가능성이 있었다.

이후에는 HSV와 LAB 두 가지 색공간을 중심으로 비교 실험을 진행했다.

---

## 4. HSV와 LAB 비교 실험
색상 기반 전처리 실험에서는 HSV와 LAB를 모두 사용해 보았다.

### HSV 방식
- Hue, Saturation, Value를 기준으로 마스크를 생성했다.
- 색상 범위 지정이 직관적이고, 특정 색에 대한 검출에 유리했다.
- 다만 밝기 변화에 민감하거나, 색상이 비슷한 환경에서는 잡음이 생기기 쉬웠다.

### LAB 방식
- L(밝기), A(빨강/초록 축), B(파랑/노랑 축)로 색상을 분리한다.
- 밝기와 색상 성분을 분리해서 처리할 수 있어, 조명 변화에 상대적으로 덜 민감했다.
- 라인의 색상과 배경의 색상 차이를 더 안정적으로 분리하는 데 유리했다.

### 실험 결론
- HSV는 특정 색상 검출에는 편리했지만,
- LAB는 이 프로젝트의 목적에 더 잘 맞는 방식으로 인식되었다.
- 특히 밝기 변화가 있는 환경이나, 라인과 배경의 대비가 색상 차이로 드러나는 상황에서는 LAB가 더 안정적이었다.

즉, 이 프로젝트에서는 "색상 기반이 필요하다"는 방향은 확정되었고, 그중에서도 LAB가 더 우선적인 후보로 사용되었다.

---

## 5. 라인 검출 파이프라인 구축
색상 마스크를 만든 뒤, 이를 바탕으로 실제 라인을 검출하는 파이프라인을 구성했다.

### 기본 흐름
1. 컬러 프레임 입력
2. HSV 또는 LAB 마스크 생성
3. morphology 연산(Closing/Open)으로 노이즈 제거
4. Canny 엣지 검출
5. Hough / HoughLinesP로 직선 후보 검출
6. RANSAC으로 노이즈 제거 및 안정적인 직선 fitting

### 실험한 방식들
- Canny + Hough
- Canny + HoughLinesP
- HoughLinesP + RANSAC
- 기울기 기반 가로/세로 선 분류
- 좌/우 선분 분리 후 각각의 라인 추정

### 주요 목적
- 단순히 선이 보이기만 하는 것이 아니라,
- 실제로 유의미한 선 하나를 안정적으로 탐지하고,
- 그 선을 기반으로 위치를 추론할 수 있게 만드는 것이 목표였다.

---

## 6. RANSAC을 활용한 선 안정화
허프 변환으로 검출된 선분들은 노이즈가 섞이기 쉬웠기 때문에, RANSAC을 사용해 가장 잘 맞는 직선을 찾는 실험을 진행했다.

### 왜 RANSAC을 사용했나
- 허프 변환은 선분 후보를 많이 뽑지만,
- 그 중 실제 라인과 무관한 노이즈도 포함된다.
- RANSAC을 섞으면 실제 라인에 맞는 점들만 남겨서 보다 안정적인 직선 추정이 가능하다.

### 실험 포인트
- threshold 값 조절
- iteration 수 조절
- 선분 점들의 밀도와 품질에 따라 fitting 성능 차이를 확인

이 과정은 단순 시각 검출보다 위치 추정에 더 적합한 구조를 만드는 데 중요한 역할을 했다.

---

## 7. 위치 추정 및 네비게이션 관련 실험
라인 검출 결과를 기반으로, 카메라만으로 위치를 추정하려는 다양한 접근을 시도했다.

### 7.1 가로/세로 선 분류
- 선의 기울기를 기준으로 가로선과 세로선을 구분했다.
- 이를 통해 격자 형태의 구조를 더 잘 인식하려고 했다.

### 7.2 좌/우 라인 분리
- 영상 중앙 기준으로 왼쪽/오른쪽 라인을 나눠서 처리했다.
- 각 영역에서 독립적으로 선을 모델링하고, 이를 기반으로 위치를 갱신하려고 했다.

### 7.3 격자 기반 좌표 추정
- 선이나 교차점을 밟을 때마다 좌표를 증가시키는 방식으로,
- 전역 위치를 간단한 격자 좌표로 추정하는 실험을 진행했다.
- 즉, "라인을 따라 이동하면 위치가 증가/감소하는 구조"를 만드는 방식이다.

### 7.4 게이팅 기반 방식
- 격자 간격을 학습하고,
- 다음 노드/교차점에 도달했는지 판단하는 게이팅 방식도 시도했다.
- 이 방식은 위치 보정과 노드 인식에 유용할 것으로 기대되었다.

---

## 8. 노드/교차점 인식 실험
교차점 또는 노드를 감지하는 실험도 진행했다.

### 접근 방식
- 수평선과 수직선이 동시에 많이 나타나는 구간을 노드로 간주
- ROI를 설정해 화면 중앙부에서 교차 구조를 확인
- HoughLine 결과를 바탕으로 노드 여부를 판단
- 분기점/교차점이 나타날 때 좌표를 갱신하거나 상태를 전환하는 방식 사용

### 실험 목표
- 단순한 라인 추적이 아니라,
- 격자 구조에서 특정 지점(노드)을 인식해 위치 기준점으로 삼는 것이다.

---

## 9. ArUco 마커 활용 실험
라인 기반 추정 외에도, ArUco 마커를 이용한 시각적 기준점 인식 실험도 함께 진행했다.

### 사용 목적
- 실제 세계의 특정 위치를 시각적으로 표시하는 기준점으로 사용
- 라인 기반 추정 결과를 보정하거나 검증하는 용도

### 구현 내용
- ArUco marker detection
- marker ID와 중심 좌표 저장
- detected_markers를 통해 인식된 마커 목록 관리

### 의미
- ArUco는 라인 기반 위치 추정의 보조 수단으로 매우 유용하다.
- 다만 단독으로 전체 위치 추정을 완성하기에는 제한이 있으며,
- 라인 기반 구조와 함께 쓰는 것이 더 자연스럽다.

---

## 10. 실험에 사용된 주요 파일
다음 파일들을 중심으로 실험이 진행되었다.

- t1.py: 흑백 기반 이진화 및 기본 라인 실험
- line_detect.py: Hough 기반 라인 검출 실험
- line_detect2.py: HoughLinesP + RANSAC 기반 실험
- line_detect2_wh.py: 가로/세로 선 분류 실험
- line_detet2_xy.py: 좌표 추정 및 위치 갱신 실험
- color_seg.py: HSV/LAB 색상 마스크 비교 실험
- color_gating.py: 게이팅 기반 위치 갱신 실험
- color_hough.py: 색상 기반 노드/교차점 감지 실험
- color_line_length.py: 분기점/노드 인식 강화 실험
- color_final2.py: LAB 기반 라인 추출 + ArUco 인식 통합 실험
- canny.py, ransac.py: 공통 전처리/선형 fitting 모듈

---

## 11. 실험 과정의 핵심 정리
이 프로젝트에서 가장 중요한 흐름은 다음과 같다.

1. 처음에는 흑백 기반의 단순한 선 감지로 시작했다.
2. 그러나 라인과 배경의 대비가 밝기만으로 충분하지 않다는 한계를 느꼈다.
3. 그래서 컬러 기반 접근으로 확장했고,
4. HSV와 LAB를 비교하면서 LAB가 더 안정적인 방향이라는 점을 확인했다.
5. 이후 색상 마스크 → Canny → Hough/HoughLinesP → RANSAC의 조합으로 라인 검출 성능을 높였다.
6. 이를 바탕으로 위치 추정, 노드 인식, 격자 기반 좌표 갱신, ArUco 보조 검증까지 확장했다.

---

## 12. 최종적으로 정리한 방향성
현재 이 프로젝트의 방향은 다음과 같이 요약할 수 있다.

- GPS 없이 카메라만으로 위치를 추정하는 문제에 대해,
- 라인 기반 시각 추정 방법을 중심으로 접근했다.
- 흑백 기반보다 컬러 기반이 더 적합했고,
- 그중 LAB 기반 마스크가 가장 안정적인 실험 방향으로 확인되었다.
- 라인 검출은 단순 허프 변환보다, 색상 마스크 + Canny + HoughLinesP + RANSAC 조합이 더 실용적이었다.
- 위치 추정은 단순한 라인 추적을 넘어, 교차점/노드/격자 구조를 이용한 좌표 갱신으로 확장되었다.

이 실험들은 향후 실제 UAV 내비게이션에 적용하기 위한 기초 검증 단계로 볼 수 있다.
